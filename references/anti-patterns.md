# 因子正交化反模式

| 反模式 | 为什么错 | 修复 |
|---|---|---|
| 把 forward return 放进控制变量 | 直接泄漏 label，残差因子不可用 | 控制变量只能是 T 日已知暴露 |
| 用全样本 pooled 回归 | beta 混入未来截面结构 | 必须逐日截面回归 |
| 用未来行业分类回填历史 | 历史行业暴露被污染 | 使用当时点行业分类 |
| 正交后不重新 z-score | 残差尺度漂移，IC / 组合权重失真 | 每日残差重新标准化 |
| 只看正交后 score | 忽略独立性目标 | 同时看暴露下降和 alpha 保真 |
| 控制变量过多 | 小截面回归不稳定，残差变噪声 | 控制变量数量与样本数匹配 |
| 与旧因子相关性仍大于 0.8 | 不是新信息，只是旧因子变体 | 加入旧因子矩阵或拒绝 |
| 用 test 段挑正交方案 | 污染最终评估 | train/val 调参，test 严格不可见 |
| 忽略 coverage 变化 | dropna 后样本悄悄缩水 | 报告正交前后 coverage |
| 忽略 turnover | 残差化可能放大噪声 | 报告 turnover delta |

## 危险信号清单

如果出现以下情况，先暂停，不要直接接受残差因子：

- 正交后 rank IC 比原始因子更高很多
- 行业 / 市值 exposure 没降，但 score 变高
- coverage 下降超过 10 个百分点
- annual turnover 翻倍
- old factor max correlation 仍然大于 0.8
- 残差因子在 train / val / test 三段表现几乎一样好

真实正交通常会牺牲一部分 IC。完全不损失甚至变强，需要先排查泄漏、mask 错位和回归实现。
